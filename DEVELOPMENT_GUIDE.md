# Development Guide — Echem GUI

---

## Stage 1: First Time on a New PC

Do this once per machine.

### Step 1 — Install prerequisites
- [Python 3.12+](https://www.python.org/downloads/) — check "Add to PATH" during install
- [Git](https://git-scm.com/downloads) — use default options
- [PyCharm](https://www.jetbrains.com/pycharm/) — Community edition is fine

### Step 2 — Clone the repository
Open a terminal and run:
```bash
cd C:\Users\YourName\PycharmProjects
git clone https://github.com/M-Song-ChE/echem-gui.git
```
This creates the `echem-gui/` folder automatically. Do not create it manually.

### Step 3 — Create a virtual environment
```bash
cd echem-gui
python -m venv .venv
.venv\Scripts\activate
```
You should see `(.venv)` appear at the start of your terminal line.

### Step 4 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 5 — Open in PyCharm
- Go to `File → Open` → select the `echem-gui` folder → click **OK**
- ⚠️ Use **Open**, not **New Project**
- Set the interpreter: `Settings → Project → Python Interpreter → Add → Existing → .venv\Scripts\python.exe`

### Step 6 — Verify it runs
```bash
python run_echem.py
```
The GUI window should open.

## Stage 2: Further Development (Returning to the Project)

Do this every time you sit down to work after the initial setup.

```bash
cd C:\Users\YourName\PycharmProjects\echem-gui

# Activate the virtual environment
.venv\Scripts\activate

# Pull the latest changes from GitHub before touching anything
git pull origin main
```

If new packages were added by someone else (or yourself on another PC), reinstall:
```bash
pip install -r requirements.txt
```

## Stage 3: Daily Routine

### Start of every session
```bash
.venv\Scripts\activate
git pull origin main
```

### End of every session
```bash
# Stage only the files you changed (safer than git add .)
git add echem_core/app.py echem_core/plotting.py   # example

# Commit with a clear message
git commit -m "Fix axis label for J density column"

# Push to GitHub
git push origin main
```

### If you changed multiple files at once
```bash
git status                  # see what changed
git add <file1> <file2>     # stage specific files
git commit -m "your message"
git push origin main
```

## Requirements File Tips

`requirements.txt` lists every package the project needs to run.

**When you install a new package:**
```bash
pip install some-package
```
Then update the requirements file manually by adding:
```
some-package>=x.y.z
```
Or regenerate the full list automatically:
```bash
pip freeze > requirements.txt
```

**On the other PC after pulling:**
```bash
pip install -r requirements.txt
```
This installs any new packages that were added since the last session.

**Best practice:** use `>=` (minimum version) rather than `==` (exact version) so the project
stays compatible across different machines without being too rigid.

Example `requirements.txt` for this project:
```
pandas>=3.0.0
numpy>=2.4.0
matplotlib>=3.10.0
openpyxl>=3.1.0
```

## .gitignore Tips

`.gitignore` tells Git which files to never track. This project already ignores:

| Ignored | Reason |
|---|---|
| `.venv/` | Each PC creates its own — never commit it |
| `__pycache__/`, `*.pyc` | Auto-generated bytecode — not needed |
| `.idea/` | PyCharm machine-specific settings |
| `.claude/` | Claude Code local session data |
| `*.xlsx` | Output files — regenerate from the app |
| `Thumbs.db`, `.DS_Store` | Windows/macOS OS metadata |

**If you accidentally stage a file that should be ignored:**
```bash
git rm --cached <filename>      # removes it from Git tracking without deleting the file
git commit -m "Remove accidentally tracked file"
```

**If you want to ignore a new file type going forward**, add a line to `.gitignore`:
```
*.log
data/
```
Then commit the updated `.gitignore`:
```bash
git add .gitignore
git commit -m "Update gitignore"
git push origin main
```
